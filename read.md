# Business Scenario 
TechCare Solutions receives 200+ IT support requests weekly via email and phone. No centralised system exists, leading to missed SLAs, duplicated work and frustrated staff.

# Business Requirements
- Centralized ticket logging system 
-	Automatic ticket creation from email or by logging into the app by the customer
-	Ticket categorization and prioritization 
-	SLA tracking and alerts 
-	Ticket assignment to Agents (manual & automatic) 
-	Status tracking (Open, In Progress, Closed) for staff. 
-	Notification system (email or Teams) 
-	Dashboards for all roles i.e. Staff, Agents and Tech Agents

# Columns
*************************
## Ticket

  **TicketDate datetime**                       - Holds the date ticket was logged<br>
  
  **Ticket_ID text [primary key]**          - Holds the autogenerated ticket_id as soon as the request is being made

  **Agent_ID text**                         - Agent handling the request

  **staff_id text**                         - staff making the request

  **Computer_ID text**                      - 

  **Urgency text**                          - Speaks to the priority of the request

  **Problem_Category_id text**              - This speaks to problem category related to the problem

  **Problem_Description text**              - Detailed description of the request
  
  **Upload_ScreenShot imagelink**           - links to images that shows error messages et al (if any) for the problem to aid the tech agents understanding of the problem
  
  **Days_to_Resolved integer**              - holds the proposed days to resolve issues
  
  **Resolved bool**                         - This is a Yes/No bool, showing if the problems is resolved.
  
  **ResolvedDate datetime**                 - If the above returns Yes then timestamp is returned, if No and is above the days to resolve, Escalation is needed 
  
  **Escalated bool**                        - This is a Yes/No bool, if there is need for escalations
  
  **EscalationDate datetime**               - If escalation returns Yes, a timestamp is recorded. 

## Staff
  **staff_id text [primary key]**           - holds assigned staff id
  
  **FirstName text**                        - First Name of staff making request
  
  **LastName text**                         - Last Name of staff making request
